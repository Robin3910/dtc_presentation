INSIGHT · 核心认知

命令不重要，
目的才重要

对普通人来说，记住 Claude 的各种命令、目录结构、编排方式——没那么重要。

了解 Agent 能给我们完成哪些工作，达成哪些目的，才是最关键的，也是当下 ROI 最高 的。

PRINCIPLE · 设计原则

核心：功能拆得
足够细、足够精

也就是程序设计中的

高内聚

每个 Agent 专注做好一件事，功能高度集中

低耦合

Agent 之间依赖最小，方便单独修改和替换

细拆功能
→
高内聚
→
低耦合
→
可复用
→
易组装

方便后续复用和组装，是 Agent 工作流长期运营的基础
