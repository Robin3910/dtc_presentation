## Part 1：从对话到 AI 员工

### 自我介绍

我叫曾洛镔，之前是 vivo 的 C++ 服务器工程师，非常感谢南希哥本次的邀请。虽然我不是这个行业的专家，但希望今天可以通过程序员的角度，给大家带来一些关于 Agent 的思考和分享。

### 与 Agent 的初遇

我最早开始接触 Agent 应该是在 24 年 8 月，当时第一次接触的 Agent 就是 Cursor。

当时他还不叫 Agent，只是一个代码编辑器，只不过他可以通过和我的对话，直接修改我的代码，我当时觉得十分神奇，没想到大模型已经可以做到这个地步了。

### Agent 的原理

其实后来回想一下，原理也比较简单——不是说大模型的原理，而是指 Agent 的原理。大家可以想象一下：

> Agent 就是提前封装了一些提示词，在打开代码编辑器的时候，他主动去获取了我们文件的目录结构，并提前读好了我们的文件内容（代码）。在我和他对话的过程中，他就直接按照我们的意思去修改我们的文件，最终达到修改代码的目的。

此外，他还能帮我执行很多黑框框命令，用来做调试和测试。对于程序员来说，就好比一个非常好用的帮手。

在那之后，我基本不自己动手写代码了，Cursor 帮我解决了 70% 左右的工作，我只负责点检功能和测试，然后反馈给他，让他再继续给我改，改到我满意为止。

### Agent 的本质

其实，Cursor 代码编辑器就是 Agent 的雏形。Agent 相比于市面上最普遍的问答 AI，最本质的区别就在于，他可以直接操控电脑，通过黑框框命令行来编写程序脚本、控制文件夹、联网搜索等等，基本上涵盖了我们电脑上 90% 的操作。

**那这意味着什么呢？** 意味着我们很多日常工作都可以交给 Agent 去做：
- 写代码
- 作图
- 写文案
- 发布文章
- 发布视频

等一系列能想到的工作。

### 市面上常见的 Agent 工具

现在市面上的 Agent 工具，说白了分三派：

| | **成品化应用** | **编排框架** | **产品工具型** |
|---|---|---|---|
| **代表产品** | OpenClaw | LangGraph | Claude Code、Codex |
| **产品形态** | 开箱即用的完整应用 | SDK / 库（面向开发者） | CLI / IDE |
| **是否需要写代码** | 否 | 是 | 否 |
| **安装方式** | npm / Docker | pip install | npm install -g |
| **核心亮点** | 全渠道 IM 接入（25+平台）、本地优先数据不出门 | 有向图编排工作流、持久化断点恢复、Human-in-the-loop | Codex：沙箱隔离安全、MCP扩展；Claude Code：Sub-agent并行、Skills系统、MCP双向通道 |
| **开源规模** | OpenClaw 368k ★ | LangGraph 31.2k ★ | Claude Code 120k ★、Codex 80k ★ |
| **价格** | 开源免费 | 开源免费 | Claude Code：$20/月（Pro）或按用量；Codex：含在 ChatGPT Plus 中 |
| **适合场景** | 个人 / 团队日常助手，IM 里直接下指令 | 企业自研复杂多智能体，嵌入现有产品 | 日常编码、终端自动化，让 AI 真正替你干活 |

大家用得多了就会发现，其实底层非常看模型的能力。优秀的模型，其思考的深度相差甚远，最终也就决定了解决问题的质量。

从自主性来看：
- **Claude Code**：最激进，给个目标它真能自己跑通
- **Codex**：需要你驱动它
- **OpenClaw**：更像一个听话的 IM 助手

## Part 2：Agent 工作原理拆解，如何快速上手 Agent

> 一旦你学会了 Agent，你发现你就离不开它了。没有了它你会直接摆烂，因为前后工作效率的对比差别太大了，完全接受不了！

### 学习门槛

有系统了解过 Claude 的同学，刚开始学习的时候应该都会有这种感觉：怎么这么多要记住的概念，还需要记忆 Claude 的各种命令、目录结构、如何编排。

其实这对普通人就有一定的门槛了，面对这个黑框框，很多人容易望而却步，这也是我今天给大家推荐 Cursor 的原因——纯图形界面操作，基本的日常工作事项流程的开发都能满足条件。

### 核心心法

其实 Claude 的这些东西对于普通人来说，没那么重要。**能了解它能给我们完成哪些工作，达成哪些目的，才是最关键的，也是当下 ROI 最高的。**

你完全可以让 AI 帮你构思架构，你自己来设计文件夹的层级，并出一份说明书，专门解释每个文件夹是做什么的。

最终，每个文件夹都可以成为你的一个流程员工，需要做什么的时候找到对应的文件夹（Agent）去做就可以了。

你可以组装 A+B+C Agent 来完成一个任务。

### 效果与愿景

最终达到的效果就是，拼乐高积木一样。今天有新的任务要完成，可能以前没系统地搭建过这个任务的流程，如果未来经常要用到，那就可以想一想自己有哪些 Agent 员工可以用，让他们组合起来帮我完成一系列任务，达成目的，我做检查。

如果某些功能还不能满足条件，完全就可以通过自己来造（准确地说是通过引导 AI 来帮我制作工作流程），然后最终去达到目的。

### 核心原则

> **核心点**：将 Agent 的功能拆得足够细，足够精美。也就是程序设计中的高内聚和低耦合，方便后续复用和组装。

### Skills 生态

(这部分要重点讲一下)
介绍一下 Skill，如何查找一些好用的 Skill

首先是skill开源网站：GitHub、Skills.sh、Skillhub、clawhub.ai

大部分基本的工作流程，在网上都有一些优秀实践案例，很多时候不需要自己去造轮子。有要解决的问题，第一时间去网上找开源的案例，大部分情况，会需要在开源的基础上做一些修改，来满足我们的特定场景。

**学会直接用别人造好的轮子来为我们所用，非常重要，可以节约很大一部分时间。**

### 我的感悟

我经常会有一些奇奇怪怪的点子，一旦有了一些创造性的想法和点子，我一般就会马上入手去做。我非常享受这个创造的过程，我觉得很有意思，不管他最终是否能给我带来收益。久而久之，自己的动手能力在这个过程中就得到了锻炼，其实也就是解决问题的能力得到了提升。

## Part 4：如何组建团队自用的工作流

### Git 的重要性

Git 要用好，非常关键，大家可以共用，可以避免重复造轮子。

### Token 成本

日常使用过程中，消耗 Token 的量级是巨大的。所以，有便宜好用的渠道，非常重要。也决定了最终的效果是否能足够好。

### 团队协作

每次更新新的内容到团队协作空间之后，都可以让 AI 去针对新的内容去完善一下使用说明。后续的新人进来之后，就完全按照使用说明进行操作，只需要进行些许的培训，他就能上手使用 AI，并且一个人可以顶好多人用。因为 AI 大大提效了。

### 行业趋势

从软件公司的角度，也是这样的。现在一个程序员可以干以前 3-5 个程序员的工作，效率相当恐怖。
